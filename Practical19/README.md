## Task
* Implement APIs for registration and login using Authentication with Identity using the connected database (Users table). Also, implement validations using data annotations.
* Implement views for user registration and login operations through the database.
* Configure role-based authorization (Roles table) through an admin user.
* Explore authorizing and controlling access and view of functions based on the role of the accessing user.


## Note

* Task is implemented under Student Controller, Auth Controller, Service, Repository and Views.
* The normal user can only view the Student list and details of particular student but can not create, edit or delete a new student record.
* Admin user can perform all operations on student records.
